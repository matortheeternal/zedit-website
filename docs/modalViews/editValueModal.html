The edit value modal allows you to change the value of a record field/element.

The edit value modal can be opened by:

  1. Double clicking a field/element in the Record View.
  2. Right-clicking a field/element and clicking the Edit option.
  3. Pressing Enter with a field/element selected.

Applying the value

You can click the OK button or press Ctrl + Enter to apply the value to the field you edited and close the modal.

Some value types have validation. If a value is invalid, the input will have a red border, the OK button will be disabled, and pressing Ctrl + Enter will not apply the value.

Discarding the value

You can discard your changes to the value by clicking the Cancel button or by closing the modal.

Types of values

String Values

String values and single line text values, such as Editor IDs and FULL Names.

Text Values

Text values are multi-line string values, such as book descriptions. Some text values support HTML and CSS, and use a CodeMirror editor with syntax highlighting.

Number Values

Number values are integer or floating point values. Number values have basic validation.

Reference Values

Reference values are form IDs corresponding to other records in the plugin or its master files. When editing reference values you are first required to provide a record signature to reference which is be limited to valid references for the field. You can then enter any substring contained in the editor ID, hexadecimal form ID, or FULL name of the record you want to reference to search for it. Search results will appear in a dropdown. You must select a search result from the dropdown or provide a full hexadecimal form ID to apply a reference value properly.

Color Values

Color values are element structures which have a red, green, and blue value. The color editor allows you to specify a color value via an HSL gradient map or by entering an RGB/Hex color value into the text input. E.g. rgb(0, 128, 255) or #0080FF. Hex strings get converted to RGB strings once entered.

Enumeration Values

Enumeration values are integer fields with a small set of valid values which have unique meanings. E.g. The Armor Type field on armor records supports the values 0, 1, and 2, which indicate Light Armor, Heavy Armor, and Clothing, respectively. Enumeration values can be selected from a dropdown.

Flag Values

Flag values are fields where each bit indicates whether or not a particular flag is enabled. Flags are displayed as a checkbox list.

Byte Array Values

Byte array values are binary fields. They can be edited from a minature hex editor which allows you to overwrite individual bytes without changing the fixed length of the byte array.
