Step 1: Prepare

Before you do start designing your module, you should familiarize yourself with zEdit's APIs.

Prepare a development environment for creating your zEdit module. I highly recommend starting with the zEdit Module Boilerplate.

You'll want to choose a text editor or IDE as well. Below are some options:

  • Webstorm - A fully featured IDE from JetBrains. It may seem a bit overkill, but some of the features are really nice to have.
  • Atom - A highly customizable, open source text editor.
  • Visual Studio Code - Super lightweight text editor with extension support.
  • Sublime Text - Freemium text editor.
  • Notepad++ - Classic text editor, limited feature-set.

Step 2: Design

Make a plan for what your module will do in plain English. You can spend as much or as little time on this step as you want, but spending a more time on design can help you avoid time-consuming mistakes during the implementation step.

Below are some tips for designing modules:

  • Use top-down design. Break the task into smaller tasks until each task can be completed in a few lines of code.
  • Keep it simple. If something seems too complicated, it probably is.

Step 3: Implementation

Write the code. It's best to implement things iteratively, testing whenever you can.

Below are some tips for implementing modules:

  • Don't repeat yourself. Use functions and data structures when applicable. You will produce better, easier to debug code.
  • Avoid premature optimization. Code the simplest solution, then optimize as necessary.
  • Don't be afraid to ask for help.
