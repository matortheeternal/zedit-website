zMerge is the successor to Merge Plugins, and offers a variety of improvements.

Simplified Workflow

zMerge loads plugins when building merges and unloads them immediately afterwards. This means you can build any number of merges covering any number of plugin files as many times as you want without having to restart the program as long as you don't exceed 255 modules in the load order for any individual merge.

zMerge doesn't care about plugin errors or non-contiguous plugins. You no longer need to fix errors in plugins, create a custom Mod Organizer profile, or change your load order prior to creating a merge.

Improved Form ID Renumbering

zMerge compacts form IDs when renumbering, so you will never overflow your form ID space from merging too many times. zMerge also has a form ID consistency feature, so adding new plugins to or otherwise updating an existing merge will not break your save games.

Improved Integrations

zMerge suppots MO2 instances out of the box and has integrations for disabling plugins or mods automatically when merges complete successfully.

Improved Asset Handling

Added asset handling for INI files and LOD Billboards. The new asset handling system allows new asset handlers to be added cleanly and easily.

In addition, the data tab of the Edit Merge Modal allows users to preview assets detected by zMerge and configure how they will be handled per-merge instead of globally.

Improved Logging

Logs are now actively written to disk while the program executes and are more verbose without making the progress log needlessly verbose as well or requiring debug settings.

Improved Asset Copying

No more batch script copying and a lower chance of running into file system permission issues.

Planned Improvements

Other planned improvements to zMerge include the following:

  • String file merging: For merging plugins that use .DLSTRINGS or .ILSTRINGS files.
  • Merged BSA creation: Similar to the option provided in Merge Plugins, except it will probably be made using Bethesda's archive.exe instead of BSAOpt.
  • Merge upwards: This will allow you to merge upwards, e.g. into a master file.
  • FO4 MCM file handling: Fallout 4's MCM system is different from Skyrim's and involves considerably more file references. A handler for these FO4 MCM files will be added in the near future.
  • Fallout 3/NV self-reference handling: This is the same as the option provided in Merge Plugins.
  • Rebuilding NAVI: This feature will allow zMerge to rebuild NAVI records automatically when merging multiple plugin files that modify the same NAVI records so you don't have to open the CK to rebuild them.
  • Integrated Conflict Resolution: An integration to resolve conflicts for a merge through zSmash will be developed once zSmash is built.
