Feature Comparison

Feature Description
Theming zEdit allows for custom application and syntax themes via CSS where xEdit only allowed for custom conflict colors.
Modules zEdit introduces a module system, which allows developers to extend or add functionality to zEdit. xEdit has no such system, limiting larger contributions to the main codebase or standalone applications.
Language xEdit is a pure Delphi application where zEdit is an Electron application built with JavaScript, HTML, and CSS which taps into the xEdit codebase via a Delphi DLL file. The developer community for JavaScript is larger and more active than the developer community for Delphi.
Performance zEdit offers native Delphi performance by calling xEdit DLL functions. Techniques such as UI virtualization and data caching are used to keep the GUI as fast and efficient as possible.
Documentation zEdit ships with official documentation where xEdit has various user-created guides scattered across the internet.
Game mode support Both xEdit and zEdit support the same games. zEdit uses xEdit's codebase internally, so it will always be able to support the same games as xEdit.
Game mode selection In xEdit game mode selection is done either via a command line parameter or renaming the executable. zEdit allows you to select the game mode from the Start View, and will support command line parameters in the near future.
Application modes zEdit currently supports the most used "Edit" mode and a "Clean" mode for cleaning errors and dirty edits in plugin files. LOD Generation and Translation tools may be added to zEdit in the future. zEdit will provide dedicated zMerge and zSmash application modes in the near future. zEdit allows you to select the application mode from the Start View, and will support command line parameters in the near future.
Load order selection zEdit's Load Order Modal offers several UX improvements over xEdit's load order modal including multi-selection, extended hotkey support, disabling of unloadable plugins, what you see is what you get, and coloring combined with tooltips for disabled plugins and required masters.
Plugin loading zEdit doesn't build references by default. This results in much faster loading times compared to xEdit assuming you didn't use the hidden feature where you can hold shift to disable building references. In addition, zEdit has a loading spinner which displays while plugins are loading instead of showing a disabled interface.
Layout Unlike xEdit, zEdit offers a dynamic layout which allows you to create new views in tabs. In the near future you will be able to split panels, move tabs between panels, save layouts, and more.
Tree view zEdit's Tree View operates almost identically to xEdit's navigation view. Unlike xEdit's navigation view zEdit's tree view supports custom columns, can optionally display group signatures, and can optionally hide file header nodes.
Record searching zEdit offers the same record searching options as xEdit as well as the ability to search for records by FULL Name. You can press Ctrl+F on the tree view to open the record search bar.
User scripts zEdit allows for the execution of user scripts similar to xEdit via the Automate Modal. Unlike xEdit scripts, zEdit scripts are written in ES6 JavaScript and use the xelib API. Script execution is often several orders of magnitude faster than it is in xEdit. This is largely due to the V8 JavaScript engine being vastly superior to the jvInterpreter.
Error checking zEdit's zClean application mode provides a streamlined experience for checking plugins for errors and fixing them. It provides automatic resolution options for more error types than xEdit, and allows users to manually choose from multiple resolution options when cleaning plugins.
Record copying zEdit offers the same record copying functionality as xEdit as well as additional ease of use through hotkeys and reference swapping when copying multiple records.
File creation In xEdit, the only way to create a new file is from a script or by copying a record. zEdit offers a context menu option for creating new files as well as an option to create a new file when copying a record.
File renaming zEdit allows you to rename loaded files, which xEdit is unable to do.
Record view zEdit's Record View operates almost identically to xEdit's record tab. Unlike xEdit's record tab, zEdit's record view allows you to collapse and expand nodes, making working with large records easier. In addition, opening a record is done by double clicking it or dragging and dropping it in the view.
Modal value editing zEdit's Edit Value Modal is similar to xEdit's edit value modal, but improves the UX for editing reference, color, enumeration, and byte array values.
Inline value editing zEdit offers basic inline value editing on par with xEdit. Reference editing is faster most of the time because zEdit requires specifying a signature and only looks for a limited number of matching references.
Drag and drop zEdit supports drag and drop between master and override records on the record view the same as xEdit. In addition, it allows drag and drop between record views and drag and drop from the tree view to set references.
Filtering zEdit offers more extensive filtering functionality than xEdit through the Advanced Search Modal. zEdit's filtering allows users to create a flexible set of conditional statements for filtering without resorting to writing code. It also allows users to filter a scope of records, making filtering faster in practice the majority of the time.
Referenced by view zEdit's Referenced By View operates almost identically to xEdit's referenced by tab. Unlike xEdit's referenced by tab you can add custom columns to the zEdit's referenced by view.
Spreadsheet view A view similar to xEdit's spreadsheet tab is planned but not yet included in zEdit.
Messages view zEdit's Log View operates very similarly to xEdit's messages tab. Unlike xEdit's messages tab, it color-codes error and warning messages.
