The tree view displays a tree of files, groups, and records, and is primarily used for navigation.

Tree nodes

The tree view is made up of a hierarchy of nodes which represent files, groups, and records. Nodes can be expanded or collapsed by clicking on the + / - button, and you can select a node by clicking on it. See the hotkeys section below for additional interactions.

Columns

The tree view displays the same columns as xEdit by default: FormID, EditorID, and Name. You can edit columns by clicking the icon to open the edit columns modal.

Sorting

Only the three native columns: FormID, EditorID, and Name, support sorting. You can click on a column's header to sort by it or reverse its sort direction.

Hotkeys

  • Double Click - Toggles the expansion of the clicked node and opens it in the record view if it is a record or file node.
  • Ctrl + Click - Toggles the selection of the node clicked, preserving the current selection.
  • Shift + Click - Selects all nodes between the clicked node and the previously selected node.
  • Enter - Opens the selected node in the record view if it is a file or record node.
  • Right Arrow - Expands the last selected node if it is not expanded, else selects the first child node.
  • Left Arrow - Collapses the last selected node if it is expanded, else selects its parent node.
  • Up Arrow - Moves the selection up one node. Can be used for multi-selection if Shift is held.
  • Down Arrow - Moves the selection down one node. Can be used for multi-selection if Shift is held.
  • Page Down - Scrolls down one page of nodes.
  • Page Up - Scrolls up one page of nodes.
  • Delete - Deletes the selected nodes if possible.

Search bar

The search bar on the tree view can be used to search for a record by Form ID, Editor ID, or FULL Name. You can open or focus the search bar by pressing Ctrl + F.

Exact match

By default, searching by Editor ID or Name will find any record which contains the input string. You can check the exact match checkbox to find only exact matches for the input text.

Hotkeys

Note: all hotkeys only work when the search input is focused.

  • Escape - Hides/closes the search bar.
  • Enter - Starts a search to find the next record matching the search parameters.
  • Shift + Enter - Starts a search to find the previous record matching the search parameters.
  • Alt + F - Switches the search mode to Form ID.
  • Alt + E - Switches the search mode to Editor ID.
  • Alt + N - Switches the search mode to Name.
  • Alt + X - Toggles the exact match option.

Context menu

You can right-click anywhere in the tree view to open a context menu with actions. The context menu actions available are listed below:

Item Availability Hotkey Description
Add File Visible when right-clicking in an empty area on the tree view. Insert Allows you to create a new file, which will occupy the load order position immediately after the currently loaded files.
Add Group Visible when right-clicking an editable file. Insert Contains children items for each group that is not currently in the file. Click on a child item to add the corresponding group to the file.
Add Record Visible when right-clicking a group in an editable file. Insert Contains children items for each type of record which can exist in the group. Click on a child item to add a blank record of the corresponding type to the group.
Delete Visible when right-clicking a group or record in an editable file. Delete Removes the group/record from the file.
Refactor File Visible when right-clicking an editable file. F2 or Alt + Shift + R Opens a modal from which you can change the filename, author, or description of the last selected file.
Refactor Records Visible when all selected nodes are editable records. F2 or Alt + Shift + R Opens a modal from which you can change the form IDs, editor IDs, or names of all selected records.
Masters -> Add Masters Visible when right-clicking an editable file. - Opens a modal from which you can add masters to a file.
Masters -> Sort Masters Visible when right-clicking an editable file. - Sorts masters in the last selected file based on the current load order.
Masters -> Clean Masters Visible when right-clicking an editable file. - Removes unused masters from the last selected file.
Enable Editing Visible when right-clicking an uneditable file. Ctrl + E Enables editing of the file.
Automate Ctrl + M Opens the Automate Modal.
Advanced Search Ctrl + Shift + F Unimplemented.
Open Visible when the last selected node is a record or file. Enter Opens the selected node in the record view.
Copy into Visible when all selected nodes are records or groups. Ctrl + Alt + C Opens the Copy Into Modal, allowing you to copy the selected nodes into one or more files of your choice.
Copy Visible when all selected nodes are records or groups. Ctrl + C Copies the selected nodes to the application's clipboard.
Copy path Always visible and enabled. Ctrl + Shift + C Copies the paths of the selected nodes to the system clipboard.
Paste Visible when all selected nodes are valid targets for pasting the elements currently in the clipboard. Ctrl + V Pastes the nodes in the application's clipboard into the selected nodes.
Paste as override Visible when all selected nodes are valid targets for pasting the elements currently in the clipboard. Ctrl + Shift + V Pastes the nodes in the application's clipboard into the selected files/groups as override records.
